# 邏輯結構
```
1.集合結構:資料元素彼此平等,彼此屬於同一個集合
2.線性結構:資料元素是一對一的關係
3.樹形結構:資料元素之間存在一對多的層次關係
4.圖形結構:資料元素是多對多的關係
```

# 物理結構
```
資料如何存在電腦的方法
1.順序儲存結構:電腦分配一段連續位址空間來存資料,例如陣列
2.鏈結儲存結構:可以連續或不連續的空間,資料元素彼此透過指標位址來得知上一個或下一個資料元素
```

# 資料型別
```
1.原子型別:程式語言的基本型別,不可再分割,例如整數,浮點數,字元,字串等,不同語言的基本類別可能不同
2.結構型別:由多個型別組成,不一定是基本型別,例如整數陣列
```

# 抽象資料型別
```
取出事物中具有普遍性的本質
抽象資料型別(Abstract Data Type)指一個數學模型,及在該模型上的操作
例如"整數"就算一種 ADT,儘管在不同電腦類型呈現的方式可能會不同(平板,筆電,手機),但是他的數學特性是相同的
ADT 除了指基本型態外,設計師自己建立的類型也可稱為 ADT
例如有個 point 的結構,其中包含 x,y,z 整數,此 point 類型也可以稱為 ADT

ADT 並非只代表那一個定義的類型,也包含此類型與其他此類型之間的關係,以及如何操作此類型
例如: point
(1) 類型 = point
(2) point 與 point 之間的關係是一段距離
(3) point 可以和 point 相加(x1 + x2, y1 + y2, z1+z3) ...

標準格式:
ADT  型別名稱
Data 資料元素之間的定義
Operation 
     操作 1 (初始條件 / 操作結果描述)
     操作 2 (初始條件 / 操作結果描述)
     操作 n (初始條件 / 操作結果描述)
endADT     
```
